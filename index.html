Qué hace cada parte

Tres capas de un PDF, tres formas de recuperarlas en el navegador.

Jerarquía real

PDF.js expone el tamaño de fuente de cada línea de texto. Se compara contra el tamaño más frecuente del documento (el cuerpo) para asignar H2–H4, en vez de fiarse de un extractor genérico.

Tablas por columnas

Cuando varias líneas seguidas tienen huecos horizontales grandes y repetidos en las mismas posiciones, se agrupan como filas de una tabla y se vuelcan a Markdown con separador de cabecera.

OCR selectivo

Si una página tiene muy poco texto extraíble (indicio de que es una imagen o un escaneo), se renderiza a canvas y se reconoce con Tesseract.js en el idioma elegido.

El proceso, en cuatro fases

Análisis tipográfico

Se recorren todas las páginas con PDF.js para medir el tamaño de cada línea y determinar el tamaño de cuerpo del documento.

Detección de tablas

Se buscan huecos horizontales grandes y repetidos entre líneas consecutivas para reconstruir filas y columnas.

OCR de páginas con poco texto

Las páginas por debajo del umbral de caracteres se renderizan y se pasan por Tesseract.js.

Ensamblado y jerarquía

Se genera el Markdown final, remapeando encabezados repetidos a negrita cuando en realidad son etiquetas de tabla.

Honestidad sobre las diferencias con el enfoque de Colab

Tablas: aquí se detectan por alineación de columnas de texto, no por las líneas vectoriales del PDF. Funciona bien con tablas de rejilla regular; tablas muy irregulares o con celdas combinadas pueden salir imperfectas.

OCR: se aplica a la página completa cuando hay poco texto, no solo al recorte exacto de cada imagen. En páginas con mucho texto y además una infografía incrustada, el texto de esa infografía puede no recuperarse.

Todo corre en tu navegador: el PDF nunca se envía a ningún servidor. Tesseract.js y PDF.js se cargan desde CDN, pero el procesamiento del documento es local.
